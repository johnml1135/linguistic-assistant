## Why

When Gemma (or the deterministic pipeline) defers a linguistic decision — "I don't know, ask a human" — we
currently emit a one-line `defer` record. That is not actionable: a reviewer has no context, no candidate
analyses, no sense of what resolving it would change. The thin languages already produce hundreds of these
(tgl 313, swh 349 in the last sweep). We need each deferral turned into a **resolution ticket**: a
formulaic JSON+markdown package with auto-generated hypotheses, HC-verified counterfactual parses ("if A
were true this verse parses thus; if B, thus"), 5–10 scripted ways to ask the speaker, structured
resolution, and tags (domain/impact/confidence/dependencies) — fed into a bug-tracker users work one-by-one,
with resolutions flowing back into the gold via the existing `deltas/` ledger.

## What Changes

- A **strict ticket schema** (JSON + embedded markdown) and a tracked **ticket store** with a bug-tracker
  lifecycle (open → in_review → resolved/wont_fix), tags (domain, impact, confidence), and a dependency DAG
  so reviewers resolve unblocking tickets first. Resolutions write back to `deltas/`.
- An **HC counterfactual-parse engine**: a hypothesis is a grammar edit (LexEntry / MoStemAllomorph /
  MoInflAffMsa / PhonologicalRule / homograph split / segmentation); apply it to the gold grammar, re-parse
  the focus verse + 3–5 related verses, and diff the analyses. **Deterministic, no LLM.**
- A **deterministic package builder** (Phase A): enumerate hypotheses from a fixed HC-mechanism taxonomy
  keyed by deferral type, select 5–10 presentation options from `parsegym/questions.py`, compute
  impact (corpus frequency × wordforms affected) and dependencies (shared lemma/affix/stem). **No LLM
  required — the spine runs offline.**
- An **LLM enrichment layer** (Phase B): the swappable harness model proposes hypotheses the taxonomy
  misses and writes the human prose; **every model hypothesis is HC-verified before it enters the ticket**,
  so richness never costs correctness.
- A **workflow escalation** (Phase C): for tickets auto-tagged high-impact or dependency-clustered, a
  multi-agent fan-out (one hypothesis agent per mechanism, then a synthesizer) does a deeper investigation.
- New CLI to **build tickets from current `defer` records** (`propose.py` / sense_pick / affix outputs).
- A **4-stage resolution pipeline** that makes the above an optimizable loop, each stage a measurable
  signal-to-noise point: **(1) auto-accept** the low-hanging fruit (THOT ∩ a light LLM check, ≥99.5%
  precision, `source: ai-auto`, no human, auditable/revertible — no morphology/phonology); **(2) candidate
  selection** — deterministically pick the next-most-resolvable thing to attack (e.g. "these 4 surface forms
  all map to one English word → maybe one lexeme"); **(3) hypothesis generation** — THOT + algorithms (and
  optionally LLM) propose candidate grammar edits; **(4) hypothesis assessment** — HC delta + reparse +
  **regression detection** (net parse delta = gains − regressions) + LLM verdict.
- A **pipeline validation set**: hundreds of ground-truthed scenarios generated by **ablating the verified
  gold** (remove a known lexeme/affix/rule → some verses break → answer key = the removed item), typed by
  which stage they exercise, plus regression decoys — so each stage's accuracy is measured and improved
  independently before the loop is tuned to sweep more without human intervention.

## Capabilities

### New Capabilities
- `deferral-tickets`: the ticket JSON schema + markdown rendering, the tracked store, status lifecycle,
  tags (domain/impact/confidence), dependency graph, and resolution actions (accept_option /
  accept_with_words / reject_with_reason) that write back to `deltas/`.
- `counterfactual-parsing`: turn a hypothesis (grammar edit) into HC re-parses of the focus + related
  verses and a diff vs the current parse — the shared, deterministic evidence engine.
- `deterministic-package-builder`: Phase A — auto-enumerate hypotheses from the HC-mechanism taxonomy,
  pick presentation options from `questions.py`, compute impact + dependencies; LLM-free.
- `llm-package-enrichment`: Phase B — harness-model hypotheses + prose, each HC-verified before inclusion.
- `deferral-escalation-workflow`: Phase C — multi-agent fan-out for high-impact / clustered tickets.
- `auto-accept-tier`: Stage 1 — sweep low-hanging fruit at ≥99.5% precision (THOT ∩ light LLM check),
  flagged `ai-auto`, no human review, auditable + revertible.
- `candidate-selection`: Stage 2 — deterministically choose the next-most-resolvable target/hypothesis to
  attack from the THOT output + corpus algorithms (impact, resolvability, clustering of forms→one lexeme).
- `hypothesis-assessment`: Stage 4 — HC delta + reparse + **regression detection** (net parse delta) +
  deterministic scoring + LLM verdict; reject fixes that cause regressions.
- `pipeline-validation-set`: ablation-derived, stage-typed ground-truth scenarios + a per-stage scoring
  harness (precision/recall/regression-catch) so each stage's signal-to-noise is measurable and improvable.
- `language-profile`: a first-class per-language typological + operational profile (allowed affix/phon
  processes, FsFeatStruc feature space, gender-vs-noun-class, orthography, pivot, resource flags, per-language
  accuracy bar 99.5/99.9) that **prunes** the hypothesis space (no Spanish infix, no Swahili gender) and
  **configures** thresholds; features carry confidence/lock/provenance, are seeded from typology DBs
  (WALS/Grambank/Glottolog), and are **falsifiable** ("what if reduplication were allowed?" → toggle →
  ΔMDL/coverage → recommend a profile update).

## Impact

- **New code** under `research/golden/reference/` (or `research/deferrals/`): ticket schema, builder,
  counterfactual engine, store, renderer, CLI; plus skills for Phase B/C.
- **Reuses (no breaking changes)**: `golden/hc.py` (build_grammar_xml/run_parse for counterfactuals),
  `parsegym/questions.py` (the 12 presentation archetypes) + `parsegym/schema.py` (scenario shape),
  `golden/reference/propose.py` / `sense_pick.py` (the `defer` records that seed tickets), `deltas/`
  (write-back ledger), `harness/` (swappable Gemma/Opus/mock — Phases B/C only),
  **`research/assess/`** (MDL `L(G)+L(D|G)`, the scorecard, the worst-part ranking) as the authoritative
  "which hypothesis is better" metric, and `cycle/hc_phonology.py` (archiphoneme/harmony collapse).
- **Depends on** the `assess-grammar` change (its metric definitions are the contract Stage-4 consumes —
  this change does NOT redefine MDL/coverage/ambiguity/productivity; it applies them as a delta over a
  hypothesis edit and presents the deltas).
- **Language profiles**: a new `golden_sets/<pair>/profile.json` per pair, seeded from typology DBs
  (WALS/Grambank/Glottolog) where available + linguist overrides; supersedes today's `meta.writing_system`.
  Read by selection, generation, assessment, and auto-accept to prune the hypothesis space and set the
  per-language accuracy bar; uncertain features are probed via the `research/assess/` metrics.
- **Validation**: the deterministic spine has offline smoke tests; Phase B/C gated behind a live endpoint.
- **No change** to the frozen golden sets except via reviewed resolutions through the `deltas/` path.
